# Working Rules

## Core Principle

Understand deeply, land lightly.

For important product discussions in this project, the default output format should include **two versions**:

1. `逻辑版`
   Focus on essence, structure, strategy, assumptions, and deeper reasoning.

2. `轻盈版`
   Focus on the lightest realistic version that can happen now, with lower user burden and simpler execution.

## What Each Version Means

### 逻辑版

Use this version to answer questions like:

- What is the real problem underneath the surface?
- What is the structural or strategic interpretation?
- What long-term direction does this suggest?
- What are the hidden assumptions or risks?

This version should respect the user's tendency to think deeply and care about first principles.

### 轻盈版

Use this version to answer questions like:

- What is the most realistic next step?
- What is the simplest version that still feels true?
- How can this become easier to enter, easier to act on, or easier to test?
- What should be removed, softened, or deferred?

This version should protect against making the product, expression, or process unnecessarily heavy.

## Operational Rule

For each meaningful round of product discussion:

- First give the `逻辑版`
- Then give the `轻盈版`

If the topic is especially simple, the split can be short.  
If the topic is strategic or product-defining, both versions should be explicit.

## Intent

The goal is not to choose between depth and simplicity.

The goal is:

- keep depth in understanding
- keep lightness in execution

Or in one sentence:

**理解可以很深，落地必须轻盈。**

## Entry Principle

For Simpex, all user-facing entrypoints should ultimately lead to one thing:

**先开始一场真实对话。**

This is not only a copywriting principle. It is the shared entry logic for:

- product entry
- marketing entry
- brand entry

That means:

- do not first ask the user to understand the whole worldview
- do not first ask the user to carry social pressure
- do not first ask the user to complete a heavy expression
- first help the user begin one real conversation

When evaluating any page, feature, campaign, or invitation, ask:

**Does this make it easier for the user to begin one real conversation?**

If not, it may still be strategically valid, but it is probably not the right entrypoint for now.
